#  Contributing Guidelines

Thank you for your interest in contributing to the Awesome Enterprise Cybersecurity Architecture repository. We welcome contributions from the community to help maintain and expand this comprehensive resource.

##  Submission Guidelines

To ensure high quality and consistency across the repository, please adhere to the following guidelines when submitting a Pull Request (PR):

1. **Avoid Duplicates:** Before submitting a new tool, article, or resource, please search the repository to ensure it does not already exist.
2. **Correct Categorization:** Place your contribution in the most accurate directory. For example, an initial access tool should reside in `01-Offensive-Security/03-Initial-Access/`.
3. **Standardized Formatting:** Use the following Markdown format for all entries:
   * `* [Tool/Resource Name](https://link-to-resource.com) - A concise, objective description of the tool or resource and its primary use case.`
4. **Quality Assurance:** Ensure that linked resources are actively maintained, reputable, and highly relevant to enterprise cybersecurity operations.

##  Pull Request Process
* Fork the repository and create a new branch for your feature or addition.
* Ensure your commit messages are clear and descriptive.
* Submit your PR, detailing what you have added and why it brings value to the repository.
* Maintainers will review your submission and may request minor formatting changes.

We appreciate your time and dedication to advancing the cybersecurity community!
